6. Lean Canvas: СтудБаза

1 Проблема
  • HR: Затраты времени и денег на собеседования джунов без реального опыта. Оценки вуза ничего не значат."Кот в мешке".
  • Студенты: Нет мотивации писать чистый код. Нет портфолио. Знания теряются после сдачи лабы.
  • Вузы: Отрыв теории от практики, устаревшие и скучные LMS-системы. Нет самоподдерживающегося комьюнити для обмена знаниями.
2 Сегменты клиентов
  • Платящие клиенты (B2B): Технические вузы РФ.
  • Партнеры (B2G): IT-компании.
  • Пользователи: Студенты.
  • Проверяющие: Менторы (Опытные студенты / Преподаватели).
3 Уникальная ценность

«StackOverflow скрестили с Jira для вузов».

Единственная образовательная платформа, где студенты собирают верифицированное портфолио навыков (XP), код проверяется людьми, а HR видят прозрачный рейтинг талантов.

4 Решение
  • Открытая Wiki-база знаний с системой рейтинга (лайков) от комьюнити.
  • Биржа практических задач с ручным код-ревью менторами.
  • HR-Лидерборд с фильтрацией студентов по хард-скиллам.
5 Каналы сбыта
  • Студенческие IT-сообщества,хакатоны, конференции.
  • Прямые B2B продажи вузам.
6 Потоки выручки
  • Лицензия для вузов: Единовременная оплата за использование платформы.
  • White-Label: Продажа коробочного решения вузам.
  • Спонсорские задачи: Интеграция опенсурсных проектов от IT-компаний.
7 Структура расходов
  • Поддержка серверов и БД (PostgreSQL).
  • Разработка (Next.js Frontend + Java Spring Backend).
  • Маркетинг: размещение баннеров и участие в конференциях и мероприятиях, для увеличения узнаваемости продукта.
8 Ключевые метрики
  • MAU (Wiki): Активность в базе знаний.
  • Code Reviews: Количество проверенных менторами задач.
  • Success Hires: Количество офферов, сделанных HR через платформу.
9 Скрытое преимущество
  • Технологический суверенитет: Бэкенд написан на Java с использованием отечественного Axiom JDK.
  • Платформа изначально готова к финтех-нагрузкам и внедрению в энтерпрайз (Т-Банк).
